Look at the current git diff and staged/unstaged changes using `git status` and `git diff`.

Based on what changed, write a clear and descriptive commit message following this format:
- First line: short summary (max 72 chars) using a prefix like `feat:`, `fix:`, `chore:`, `refactor:`, `docs:`
- If needed, add a blank line and a brief body explaining *why* the change was made

Then:
1. Stage all relevant changed files
2. Commit with the generated message
3. Push to the remote branch
